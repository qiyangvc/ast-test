01 项目背景与问题

  • 垃圾短信/文本泛滥，传统规则匹配（关键词）误报率高、维护困难
  • 基于深度学习的文本分类成为主流方案，但面临两大挑战：
    • ❶ 类别不平衡：正常短信远多于垃圾短信，模型偏向预测多数类
    • ❷ 特征提取不足：传统 CNN/RNN 难以捕捉长距离依赖和关键信息
  • 本项目目标：构建一套高精度、模块化的垃圾文本分类系统

03 词向量训练：Word2Vec + NCE

传统 Softmax 的问题

  • 分母需遍历整个词汇表
  • 词表大时计算量爆炸
  • 训练 Loss 先降后升，不稳定

我们的方案：NCE 损失

  • 将多分类转为二分类：判断是真实上下文词还是噪声词
  • 只需采样 64 个负样本
  • 训练稳定，Loss 正常收敛
loss = tf.nn.nce_loss(weights, biases, labels, inputs, num_sampled=64, num_classes=vocab_size)

04 基础分类器实现

我们实现了三种基线模型，覆盖不同文本特征表示方式：

MLP

输入：词向量求和 (200维)

Dropout(0.5)
→ Dense(200, ReLU)
→ Dropout(0.5)
→ Dense(2)

▸ 简单快速，但完全丢失词序

CNN

输入：词向量序列 (20×200)

Conv1D(6 filters, k=3)
→ MaxPool1D(3)
→ Flatten
→ Dropout(0.5)
→ Dense(2)

▸ 提取局部 n-gram 短语特征

RNN (LSTM)

输入：词向量序列 (20×200)

LSTM(64)
→ Dropout(0.2)
→ Dense(2)

▸ 门控机制捕捉长距离依赖

05 创新点 ①：BiLSTM-Attention

双向 LSTM

  • 前向 + 后向两个 LSTM
  • 每个词同时拥有完整上下文
  • 解决单向 RNN 信息缺失问题

自注意力机制

  • 为每个词计算注意力权重
  • 自动聚焦关键词（如"中奖""转账"）
  • 加权求和得到句向量表示
# 网络结构 输入 → Bidirectional(LSTM) → SelfAttention → Dropout → Dense(2) # 自注意力计算 e_i = tanh(x_i · W + b) · u # 每个词的注意力分数 α_i = softmax(e_i) # 归一化为权重（和为1） context = Σ(α_i · x_i) # 加权求和得到句子向量

▸ 这是本项目效果最好的单模型

6 / 10

06 创新点 ②：Focal Loss

问题：类别不平衡导致模型偏向预测"正常短信"，垃圾短信漏检率高

核心思想

  • 降低简单样本（易分类的正常短信）的权重
  • 提高难样本（难分垃圾短信）的权重
  • 让模型更关注"学不会"的样本

公式

FL = α × (1 - p)ᵞ × CE

p 越高（越容易）→ 权重越小
γ=2 时，p=0.9 的权重仅剩 0.01

07 创新点 ③：训练策略优化

余弦退火学习率

  • Warmup 预热：前 10% 步数线性升温
  • 余弦退火：后续按余弦曲线降温至接近 0
  • 避免训练后期震荡，精细搜索最优解

早停法 (Early Stopping)

  • 监控验证集准确率
  • 连续 10 个 epoch 无提升则自动停止
  • 防止过拟合，节省训练时间
其他辅助策略：标签平滑 (Label Smoothing) 防止模型过度自信；Dropout 正则化；模型集成 (加权平均 / 投票) 提升稳定性

关键发现：基线 CNN 因类别不平衡导致 Recall 仅 81.5%（大量垃圾短信漏检）。引入 Focal Loss 后 Recall 跃升至 91.2%；BiLSTM-Attention 在 Precision 和 Recall 上达到最佳平衡，F1 达 95.3%
